## 🌟 ویژگی‌ها

- **معماری فوق‌العاده مدرن**: توسعه‌یافته بر بستر کامل Asynchronous با استفاده از FastAPI (برای Webhooks) و Aiogram 3.
- **پردازش امن درگاه ارزی**: اتصال مستقیم به NOWPayments همراه با سیستم تشخیص هوک‌های موازی و هندل کردن خودکار Retryها.
- **پشتیبانی کامل از X-UI Stream**: استخراج مطمئن و خودکار تنظیمات کانفیگ (VLESS, VMess, Reality, WSS, TCP, KCP, gRPC) با اعتبارسنجی قوی از طریق Pydantic.
- **مدیریت Dependency ها و جلوگیری از کندی**: پیاده‌سازی صحیح شیء ربات (Bot DI) که مانع از مصرف اضافی منابع RAM سرور می‌شود.
- **لاگینگ حرفه‌ای سازمانی**: ادغام شده با `structlog` جهت تولید لاگ‌های مبتنی بر JSON مناسب برای دیباگ و سیستم‌های مانیتورینگ.
- **بات بک‌گراند**: دارای سیستم مینی-ورکر با `APScheduler` برای چک کردن سلامت کانکشن‌ها، بازاریابی مجدد، و نوتیفیکیشن‌های اتمام حجم/زمان.
- **دیتابیس سریع و امن**: بهره‌گیری از SQLAlchemy نسخه ۲ همراه با AsyncPG و سیستم کشینگ Redis.

## 🛠 پیش‌نیازها

- یک سرور لینوکس ترجیحاً اوبونتو 20.04 یا بالاتر (رم حداقل ۱ گیگابایت).
- دسترسی `root` یا کاربری با دسترسی‌های sudo.
- یک دامنه یا ساب‌دامین متصل به آی‌پی سرور (برای دریافت اتوماتیک Webhook ها از درگاه NOWPayments).

## 🚀 نصب و راه‌اندازی سریع (One-Liner)

رنگ و بوی نصب‌های سخت رو فراموش کنید. فقط کافیه وارد ترمینال سرور خودتون بشید و دستور تک‌خطی زیر رو با دسترسی روت اجرا کنید:

```bash
wget -qO- https://raw.githubusercontent.com/sina99778/marzbot/main/install.sh | sudo bash
```

مراحل نصب کاملاً هوشمند (Interactive) است و خودش از شما Token ربات، رمز دیتابیس و آدرس پنل رو می‌پرسه، سپس تمام پیش‌نیازهای داکر رو به همراه گواهینامه SSL برای دامنه نصب می‌کنه.

### آپدیت کردن ربات از گیت‌هاب

توی سروری که ربات رو نصب کردید اگه بخواید آپدیتش کنید کافیه دستور `install.sh` اصلی که داخل لوکال مسیر `/var/marzbot` (یا محلی که کلون کردید) دانلود شده رو دوباره اجرا کرده و گزینه `6` یعنی `Smart Update Project Files` رو انتخاب کنید.

## 🏗 ساختار پروژه (معماری)

- `apps/api`: اپلیکیشن FastAPI برای هندل کردن وب‌هوک‌های درگاه و مدیریت بک‌گراند.
- `apps/bot`: هندلرهای پیام و Callback های تلگرام، نوشته شده با Aiogram.
- `apps/worker`: سرویس‌های Background و جاب‌های زمان‌بندی شده برای سینک سرور و نوتیس‌ها پرداخت.
- `services`: منطق‌های سنگین تجاری از جمله مدیریت شارژ (Wallet)، صدور کانفیگ X-UI و اعتبارسنجی در پرداخت مستقر است.
- `schemas/internal`: فایل‌های اعتبارسنجی امن تنظیمات با Pydantic.
- `core`: پیکربندی‌های کلی و تنطیمات لاگری (structlog).

## 👨‍💻 توسعه‌دهندگان

این ربات به صورت اختصاصی با تکنیک‌های Clean Architecture مدل‌سازی شده تا توان پاسخ‌دهی به ده‌ها هزار کاربر بدون کندی و نشست حافظه (Memory Leak) در AioHTTP را داشته باشد.
